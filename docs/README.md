# docs/

Long-form documentation for PrepperPi.

User-facing:
- `quickstart.md` — the plug-and-play user path.
- `hardware.md` — supported Pis, SSDs, power, enclosures.
- `architecture.md` — how the services fit together.
- `clean-room-policy.md` — contribution ground rules.
- `troubleshooting.md` — common issues and fixes.

Audit / contributor reference:
- `security-pentest-2026-04.md` — pen-test report for the admin console.
  Per-AC findings, code changes, residual risks.
- `creating-bundles.md` — schema + howto for authoring content bundles.
  Mirror of the
  [`prepperpi-bundles`](https://github.com/jmarler/prepperpi-bundles)
  README so contributors landing in either repo find the full spec.

Keep user-facing material approachable.
