# prepperpi-aria2c

Long-lived aria2c daemon used by the admin Catalog page (E2-S3) to download Kiwix ZIMs in the background. JSON-RPC over `127.0.0.1:6800`; the admin console is the only client.

## How it fits

```
  user clicks "Download" in /admin/catalog
        │
        ▼  HTTP POST /admin/catalog/queue  (admin uvicorn, prepperpi-admin user)
                │
                ▼  JSON-RPC `aria2.addUri` to 127.0.0.1:6800 with token from
                   /etc/prepperpi/aria2/secret.env
        ┌────────────────────────────┐
        │  prepperpi-aria2c.service  │  user=prepperpi
        │  aria2c --enable-rpc       │
        └────────────────────────────┘
                │
                ▼  HTTP GET (resumable, 4 connections, 2 concurrent downloads)
                   to mirror.kiwix.org / library.kiwix.org
        ┌────────────────────────────┐
        │  /srv/prepperpi/zim/         <-- internal storage (SD)
        │    └── .downloading/      <-- aria2 writes here while in flight
        └────────────────────────────┘
                │
                ▼  on-download-complete hook
                   mv .downloading/foo.zim → foo.zim
                │
                ▼  prepperpi-kiwix-reindex.path watcher fires
                   library.xml rebuilt; new tile on the landing page
```

## Why a separate service

aria2c needs to keep running across user sessions and SSH disconnects. Running it as a systemd-managed daemon under the `prepperpi` user (which already owns `/srv/prepperpi/`) means:

- Pause / resume / cancel survive a reboot via aria2's `--save-session`.
- File ownership of completed ZIMs matches what `kiwix-serve` and `prepperpi-usb` already expect.
- The admin uvicorn (running as the unprivileged `prepperpi-admin` user) doesn't need any sudoers rule for this — it just talks RPC over localhost.

## Sandboxing decision

This unit deliberately runs *without* `ProtectSystem=strict` (and the `RestrictNamespaces=true`, `ReadWritePaths=…` that go with it). The threat model is bounded by the `prepperpi` user, which only owns its own data tree, so the kernel-tunable + control-group + capability protections do enough heavy lifting on their own. Also avoids a category of mount-namespace-vs-host divergence that bit us during E2-S3 development (private namespaces don't propagate remount option changes from the host, so a USB drive toggled `rw` would appear `ro` inside the unit's namespace).

## RPC secret

`/etc/prepperpi/aria2/secret.env` holds the auto-generated RPC token, mode `0640` owned `prepperpi:prepperpi-admin`. systemd loads it as `EnvironmentFile=` and passes it to aria2c via `--rpc-secret=${ARIA2_RPC_SECRET}`. The admin uvicorn reads the same file directly to construct authenticated RPC calls. No other user can read it.

The secret is generated once on first install (`head -c 24 /dev/urandom | base64`) and preserved across re-installs so paused sessions survive `installer/install.sh` re-runs.

## Files

| Path | Role |
|---|---|
| `prepperpi-aria2c.service` | systemd unit; sandboxed, write-restricted to the ZIM trees and aria2's state dir. |
| `aria2.conf.tmpl` | Daemon-wide aria2 config (concurrency, session, hooks). |
| `on-complete.sh` | Hook script: moves completed file out of `.downloading/` to its final path. |
| `on-error.sh` | Hook script: emits a `download_failed` dashboard event. |
| `setup.sh` | Idempotent installer. |

## Deploy layout

| Installed path | Source |
|---|---|
| `/etc/systemd/system/prepperpi-aria2c.service` | `services/prepperpi-aria2c/prepperpi-aria2c.service` |
| `/etc/prepperpi/aria2/aria2.conf` | `services/prepperpi-aria2c/aria2.conf.tmpl` |
| `/etc/prepperpi/aria2/secret.env` | generated by `setup.sh` |
| `/opt/prepperpi/services/prepperpi-aria2c/{on-complete,on-error}.sh` | `services/prepperpi-aria2c/{on-complete,on-error}.sh` |
| `/var/lib/prepperpi/aria2/session.txt` | seeded empty by `setup.sh`, written by aria2 |

## Debugging

```
systemctl status prepperpi-aria2c.service
journalctl -u prepperpi-aria2c.service -f
curl --data '{"jsonrpc":"2.0","id":"x","method":"aria2.getVersion","params":["token:'"$ARIA2_RPC_SECRET"'"]}' \
     http://127.0.0.1:6800/jsonrpc
```
